| Reservation data -> log output | Reservation objects contain guest names, phone numbers; only key operational fields logged |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Information Disclosure | WebSocket message logging | mitigate | D-07 enforced: wsLog.debug logs `messageType: msg.type` only. Payload (guest names, phone numbers, reservation data) is NEVER passed to any log call. Grep verification: no `msg.reservationId`, `msg.state` content, or `raw` variable in log arguments. |
| T-05-05 | Information Disclosure | Reservation audit events | mitigate | D-08 enforced: audit events log only operational fields (date, startTime, partySize, category, status). Guest name (`guestName`), phone (`phone`), and notes (`notes`) are explicitly excluded. Update handler uses spread-conditional to log only changed fields, further reducing exposure. |
</threat_model>
