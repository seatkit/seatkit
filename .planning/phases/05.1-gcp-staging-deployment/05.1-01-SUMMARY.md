## Decisions Made

1. **Preserved monorepo directory structure in API runner** -- The plan specified a flat runner layout (`COPY dist ./dist`, `CMD ["node", "dist/index.js"]`), but pnpm's node_modules contains symlinks like `node_modules/@seatkit/types -> ../../packages/types` that only resolve if the monorepo directory structure is preserved. Changed to `/app/packages/api/dist/` layout with `CMD ["node", "packages/api/dist/index.js"]`.

2. **Separate prod-only installer stage** -- Split the original installer+builder into separate stages: installer with `--prod` flag (no devDependencies) and builder with all deps. The runner copies node_modules from the prod-only installer, reducing image size.

3. **Copy patches/ directory** -- The root package.json has `patchedDependencies` referencing `patches/better-auth-invite-plugin@0.4.1.patch`. Without copying this directory, `pnpm install --frozen-lockfile` fails. Added `COPY --from=pruner /app/patches ./patches` in both Dockerfiles.

4. **Created packages/web/public/.gitkeep** -- The web package had no `public/` directory. Docker COPY fails if the source path doesn't exist. Added `.gitkeep` to ensure the directory is always available for the Docker build.
