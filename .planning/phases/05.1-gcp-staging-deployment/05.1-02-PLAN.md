**Section 3: Cloud Run API Service (per D-08, D-09, D-11, D-12, D-18, D-19)**

```hcl
# =============================================================================
# Cloud Run — API Service
# =============================================================================

resource "google_cloud_run_v2_service" "api" {
  name     = var.api_service_name
  location = var.region

  template {
    service_account = google_service_account.cloud_run.email
    session_affinity = true   # D-09: WebSocket session stickiness
    timeout          = "3600s" # D-09: 60-min WebSocket timeout

    scaling {
      min_instance_count = 0  # D-11: scale to zero
      max_instance_count = 2
    }

    containers {
      image = "${var.region}-docker.pkg.dev/${var.project_id}/${google_artifact_registry_repository.seatkit.repository_id}/api:${var.api_image_tag}"

      ports {
        container_port = 3001
      }

      resources {
        limits = {
          cpu    = "1"      # D-11: 1 CPU
          memory = "512Mi"  # D-11: 512Mi memory
        }
      }

      # D-18: Secrets from Secret Manager
      env {
        name = "DATABASE_URL"
        value_source {
          secret_key_ref {
            secret  = var.secret_ids.database_url
            version = "latest"
          }
        }
      }

      env {
        name = "BETTER_AUTH_SECRET"
        value_source {
          secret_key_ref {
            secret  = var.secret_ids.better_auth_secret
            version = "latest"
          }
        }
      }

      env {
        name = "ADMIN_EMAIL"
        value_source {
          secret_key_ref {
            secret  = var.secret_ids.admin_email
            version = "latest"
          }
        }
      }

      env {
        name = "ADMIN_PASSWORD"
        value_source {
          secret_key_ref {
            secret  = var.secret_ids.admin_password
            version = "latest"
          }
        }
      }

      # D-19: Non-sensitive config
      env { name = "NODE_ENV";              value = "production" }
      env { name = "PORT";                  value = "3001" }
      env { name = "LOG_LEVEL";             value = "info" }
      env { name = "LOG_FORMAT";            value = "gcp" }
      env { name = "CORS_ORIGIN";           value = "https://${var.web_staging_domain}" }
      env { name = "GOOGLE_CLOUD_PROJECT";  value = var.project_id }
    }
  }

  # Allow unauthenticated access (API has its own auth guard)
  lifecycle {
    ignore_changes = [
      template[0].containers[0].image,  # Image updated by CI/CD, not Terraform
    ]
  }
}

# Allow unauthenticated invocations (public staging API)
resource "google_cloud_run_v2_service_iam_member" "api_public" {
  name     = google_cloud_run_v2_service.api.name
  location = var.region
  role     = "roles/run.invoker"
  member   = "allUsers"
}
```

Key design notes:
- `lifecycle.ignore_changes` on container image prevents Terraform from reverting CI/CD image updates. CI deploys new images directly; Terraform manages the service configuration.
- IAM bindings grant `secretmanager.secretAccessor` per-secret (least privilege), not project-wide.
- The `allUsers` invoker binding makes staging publicly accessible — the API has its own auth guard (Better Auth) protecting endpoints.
- DNS CNAME records point to `ghs.googlehosted.com.` which Cloud Run domain mapping resolves. Cloud Run auto-provisions managed TLS certificates.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Terraform state -> GCS | State may contain resource metadata (no secret values, but resource IDs) |
| Secret Manager -> Cloud Run | Secrets injected as env vars at container start |
| Public internet -> Cloud Run | Staging services are publicly accessible (API has auth guard) |
| Developer machine -> GCP | Terraform apply uses developer's GCP credentials |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05.1-04 | Elevation of Privilege | IAM service account | mitigate | Least-privilege: only `secretmanager.secretAccessor` per-secret, no project-wide roles; service account scoped to Cloud Run only |
| T-05.1-05 | Information Disclosure | Terraform state | mitigate | GCS backend with server-side encryption; state contains resource IDs not secret values; terraform.tfvars has no secrets |
| T-05.1-06 | Information Disclosure | terraform.tfvars | mitigate | tfvars contains only non-secret values (project_id, region, secret IDs); actual secret values live only in Secret Manager |
| T-05.1-07 | Spoofing | Public Cloud Run endpoints | accept | Staging services intentionally allow unauthenticated access (allUsers); API auth guard (Better Auth) protects endpoints at application layer |
| T-05.1-08 | Denial of Service | Cloud Run scaling | mitigate | max_instance_count = 2 caps scaling; Cloud Run has built-in request throttling; staging only |
</threat_model>
