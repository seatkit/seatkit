# Phase 05.2 Plan 01: Dark Mode Infrastructure & Design Token Foundation Summary

Dark mode via next-themes with class strategy, shadcn CLI initialized with 5 UI components (dialog, sheet, skeleton, tabs, sonner), home page redirect to /reservations, nav border added, and 3 components (ErrorBoundary, ConflictModal, PresenceBadge) migrated from hardcoded Tailwind colors to CSS variable design tokens.

## Changes Made

### Task 1: Tailwind config, shadcn CLI init, dependency installation
- Updated `tailwind.config.ts`: added `darkMode: ['class']`, status color tokens (confirmed/pending/seated/completed/cancelled/no-show), and `tailwindcss-animate` plugin
- Created `components.json` for shadcn CLI configuration pointing to `@/components/ui`
- Created `src/lib/utils.ts` with `cn()` utility (clsx + tailwind-merge)
- Installed 5 shadcn UI components manually (dialog, sheet, skeleton, tabs, sonner) since the shadcn registry was unreachable
- Installed dependencies: next-themes, motion, sonner, class-variance-authority, tailwindcss-animate, clsx, tailwind-merge, @radix-ui/react-dialog, @radix-ui/react-tabs
- **Commit:** `3eecf93d`

### Task 2: Dark mode infrastructure + ThemeToggle + home page redirect + nav border
- Created `ThemeProvider` wrapper around next-themes with `defaultTheme="dark"`, `attribute="class"`, `storageKey="seatkit-theme"`
- Created `ThemeToggle` component cycling system->dark->light with Monitor/Moon/Sun icons and mounted state guard for hydration safety
- Updated `layout.tsx`: added `suppressHydrationWarning` on html tag, ThemeProvider wrapping body content, ThemeToggle in nav, Toaster from sonner, `border-b border-border` on header
- Replaced `page.tsx` placeholder with `redirect('/reservations')`
- **Commit:** `98406106`

### Task 3: Fix hardcoded color tokens on ErrorBoundary, ConflictModal, PresenceBadge
- ErrorBoundary: `text-red-600` -> `text-destructive`, `bg-blue-600` -> `bg-primary text-primary-foreground`, added `role="alert"`
- ConflictModal: all `text-gray-*` -> `text-foreground`/`text-muted-foreground`, `border-gray-*` -> `border-border`, `text-amber-700` -> `text-amber-400`, added focus ring indicators on both buttons, preserved `bg-amber-600` (reserved amber use for conflict CTA)
- PresenceBadge: viewing state `bg-gray-200 text-gray-600 ring-gray-300` -> `bg-muted text-muted-foreground ring-border`, preserved editing state amber classes (reserved use)
- **Commit:** `a029d9ce`
